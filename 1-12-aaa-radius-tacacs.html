RADIUS vs TACACS+

Feature RADIUS TACACS+
Protocol UDP 1812/1813 TCP 49
Encryption Password only encrypted Entire packet encrypted
AAA Separation Combined auth + authz Separate auth, authz, acct
Command Authorization Limited Per-command authorization
Multiprotocol Limited Supports multiple protocols
Best For Network access (802.1X, VPN) Device administration
Standard IETF (RFC 2865) Cisco proprietary
âœ… When to Use Which
  • RADIUS: 802.1X, VPN authentication, wireless, general network access
  • TACACS+: Network device administration (routers, switches, firewalls)
  • Both: Many environments use RADIUS for users, TACACS+ for admins

ğŸ’¡ Interview Tip

Question: "What's the difference between RADIUS and TACACS+? When would you use each?"

Answer: "Both provide centralized AAA, but they have different strengths. RADIUS uses UDP and only encrypts the password, while TACACS+ uses TCP and encrypts the entire packet - making TACACS+ more secure for device administration. RADIUS combines authentication and authorization in one exchange, while TACACS+ separates all three AAA functions, allowing more granular control. The key difference for network administration is command authorization - TACACS+ can authorize each command individually, so you can allow 'show' commands but deny 'configure'. For this reason, I'd use TACACS+ for network device administration, and RADIUS for user network access like 802.1X, VPN, and wireless. Many enterprises use both - TACACS+ for admins, RADIUS for end users."
