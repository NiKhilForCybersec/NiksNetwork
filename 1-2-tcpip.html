TCP vs UDP

Feature TCP UDP
Connection Connection-oriented (handshake) Connectionless
Reliability Guaranteed delivery, retransmission Best effort, no guarantee
Ordering Maintains order No ordering
Speed Slower (overhead) Faster (less overhead)
Header Size 20-60 bytes 8 bytes
Use Cases HTTP, HTTPS, SSH, FTP, SMTP DNS, DHCP, VoIP, Video, Gaming
Flow Control Yes (windowing) No

Common Ports

Port Protocol Service Security Notes
20, 21 TCP FTP Unencrypted, use SFTP (22) instead
22 TCP SSH/SFTP Secure, common brute force target
23 TCP Telnet Unencrypted, use SSH instead
25 TCP SMTP Email, often blocked outbound
53 TCP/UDP DNS DNS tunneling risk
80 TCP HTTP Unencrypted web traffic
443 TCP HTTPS Encrypted web, requires SSL inspection
445 TCP SMB File sharing, ransomware vector
3389 TCP RDP Remote desktop, major attack target

âœ… Security Implications

TCP/IP security considerations:

  • SYN Flood: Attack that exhausts server resources with half-open connections
  • Port Scanning: Attackers use TCP/UDP to discover open services
  • IP Spoofing: Forging source IP for attacks or evasion
  • Session Hijacking: Predicting sequence numbers to take over connections
  • ARP Spoofing: Poisoning ARP cache for man-in-the-middle attacks
ğŸ’¡ Interview Tip

Common Question: "Explain the TCP three-way handshake."

Answer:

  1. SYN: Client sends SYN packet with initial sequence number
  2. SYN-ACK: Server responds with its own sequence number and acknowledges client's
  3. ACK: Client acknowledges server's sequence number, connection established

Security note: "SYN floods exploit this by sending many SYNs without completing the handshake, exhausting server connection tables. SYN cookies are a common mitigation."
