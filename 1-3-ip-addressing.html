IP addressing is fundamental to network communication. Understanding subnetting is essential for designing secure networks, segmenting traffic, and configuring firewall rules.

ğŸ  Street Address Analogy

Think of IP addresses like street addresses:

  • IP Address = Full address (123 Main Street, Apt 4)
  • Network ID = Street name (Main Street)
  • Host ID = House number (123, Apt 4)
  • Subnet Mask = Tells you where street name ends and house number begins
  • Default Gateway = The post office for your neighborhood - packages going outside go here first
ğŸ§® CIDR Notation Trick

For /24, /16, /8 - count by 8s from the right:

  • /8 = 255.0.0.0 (first octet = network) â†’ 16 million hosts
  • /16 = 255.255.0.0 (first two octets) â†’ 65,534 hosts
  • /24 = 255.255.255.0 (first three octets) â†’ 254 hosts
  • /32 = Single host (255.255.255.255)
ğŸ¯ Quick Math: Hosts = 2^(32-CIDR) - 2. For /24: 2^8 - 2 = 254 usable hosts (subtract network & broadcast)

Common Question: "Given 10.0.0.0/22, how many hosts can you have?"

Quick Calculation:

  1. /22 means 22 network bits, 10 host bits (32-22=10)
  2. 2^10 = 1,024 total addresses
  3. 1,024 - 2 = 1,022 usable hosts (minus network and broadcast)

Shortcut: For /24: 254 hosts. Each bit you borrow (/25, /26...) halves it. Each bit you add (/23, /22...) doubles it. /22 = 254 Ã— 4 = ~1,016 hosts.
