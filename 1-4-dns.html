DNS Security Threats

Attack Description Mitigation
DNS Spoofing/Poisoning Inject false DNS responses into cache DNSSEC, randomize source ports
DNS Tunneling Encode data in DNS queries for exfiltration DNS monitoring, block long queries
DNS Amplification DDoS Use DNS servers to amplify attack traffic Rate limiting, response rate limiting
Domain Hijacking Take control of domain registration Registrar lock, DNSSEC
Typosquatting Register domains similar to legitimate ones Brand monitoring, user awareness

Signs of DNS tunneling (data exfiltration):

  • Unusually long DNS queries (>50 characters)
  • High volume of TXT or NULL record queries
  • Queries to unusual or newly registered domains
  • High entropy in subdomain names
  • Unusual query patterns from single host

Common Question: "How does DNS resolution work?"

Structured Answer:

  1. Client checks local cache, hosts file
  2. Queries recursive resolver (ISP or corporate DNS)
  3. Resolver queries root servers â†’ get TLD referral
  4. Resolver queries TLD servers â†’ get authoritative referral
  5. Resolver queries authoritative server â†’ gets answer
  6. Answer cached based on TTL, returned to client
