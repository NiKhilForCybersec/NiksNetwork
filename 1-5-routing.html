Static vs Dynamic Routing

Aspect Static Routing Dynamic Routing
Configuration Manually configured Automatically learned
Scalability Poor (manual updates) Good (automatic updates)
Failover Manual intervention Automatic rerouting
Resource Usage Minimal CPU/memory Higher CPU/memory/bandwidth
Security More secure (no protocol) Requires authentication
Use Case Small networks, stub networks, default routes Large networks, redundant paths

Dynamic Routing Protocols

Protocol Type Algorithm Use Case AD
OSPF IGP (Link-State) Dijkstra SPF Enterprise internal 110
EIGRP IGP (Hybrid) DUAL Cisco environments 90
BGP EGP (Path-Vector) Best path selection Internet, ISP, multi-homed 20 (eBGP)
RIP IGP (Distance-Vector) Bellman-Ford Legacy, small networks 120
IS-IS IGP (Link-State) Dijkstra SPF ISP backbone 115

Administrative Distance (AD): Lower = more trusted. Connected routes = 0, Static = 1.

Routing Security

Threat Description Mitigation
Route Injection Attacker advertises fake routes Authentication (MD5), route filtering
BGP Hijacking Advertising someone else's prefixes RPKI, BGP monitoring, prefix filtering
Route Leaks Inadvertent route propagation Strict import/export policies
DoS via Routing Overwhelming routing protocol Rate limiting, authentication
âœ… Best Practice - Routing Security
  • Authentication: Enable MD5 authentication on OSPF/EIGRP/BGP
  • Prefix filtering: Only accept/advertise expected prefixes
  • Route summarization: Reduce attack surface and table size
  • Logging: Log routing protocol events
  • Control plane protection: Rate limit routing traffic
ğŸ’¡ Interview Tip

Common Question: "What's the difference between OSPF and BGP?"

Answer:

  • OSPF (Open Shortest Path First): Interior Gateway Protocol (IGP) used within a single organization. Uses link-state algorithm to find shortest path. Fast convergence. Used for internal enterprise routing.
  • BGP (Border Gateway Protocol): Exterior Gateway Protocol (EGP) used between organizations/ISPs. Path-vector protocol that considers policies, not just shortest path. The routing protocol of the internet. Used for internet connectivity and multi-homed environments.

Security note: "Both should use authentication - MD5 for OSPF areas, TCP MD5 for BGP sessions."
