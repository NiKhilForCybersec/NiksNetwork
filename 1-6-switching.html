ğŸš¨ VLAN 1 Security

VLAN 1 is the default VLAN and has special properties that make it a security risk:

  • All ports default to VLAN 1
  • Control plane traffic (CDP, VTP, STP) uses VLAN 1
  • Cannot be deleted

Best Practice: Never use VLAN 1 for user traffic. Change native VLAN on trunks. Use dedicated VLANs for management.

ğŸ’¡ Interview Tip

Common Question: "How would you prevent VLAN hopping?"

Answer:

  1. Change native VLAN: "Set native VLAN to an unused VLAN (e.g., 999), not VLAN 1"
  2. Explicit trunk config: "Manually configure trunks, don't rely on DTP negotiation"
  3. Disable DTP: "Use 'switchport nonegotiate' on trunk ports"
  4. Access ports: "Set 'switchport mode access' on all end-user ports to prevent them from becoming trunks"
  5. Prune VLANs: "Only allow necessary VLANs on trunks with 'switchport trunk allowed vlan'"
