Switching & VLANs

ğŸ“‹ What You'll Learn

Switches operate at Layer 2 and use MAC addresses to forward traffic. VLANs provide logical segmentation, which is fundamental to network security and reducing attack surface.

  • Layer 2 switching concepts
  • VLAN configuration and design
  • Trunking (802.1Q)
  • Switch security features
  • Common Layer 2 attacks
ğŸ§  Memory Aid: Switching & VLANs
ğŸ¢ Office Building Analogy

Think of a switch like an office building:

  • Switch = The building itself with multiple offices
  • Ports = Doors to each office
  • MAC Table = Reception desk directory - knows who's in which office
  • VLANs = Different floors/departments - Accounting can't walk into Engineering without going through reception (router)
ğŸ”Œ Port Types
  • Access Port = Single office door - one VLAN only, for end devices
  • Trunk Port = Elevator - carries traffic for ALL floors (VLANs) between switches
  • Native VLAN = The "default floor" for untagged traffic in trunk
ğŸ¯ Security Golden Rule: "VLANs separate broadcast domains, but a router/firewall is still needed between them for security filtering!"

ğŸš¨ VLAN 1 Security

VLAN 1 is the default VLAN and has special properties that make it a security risk:

  • All ports default to VLAN 1
  • Control plane traffic (CDP, VTP, STP) uses VLAN 1
  • Cannot be deleted

Best Practice: Never use VLAN 1 for user traffic. Change native VLAN on trunks. Use dedicated VLANs for management.

ğŸ’¡ Interview Tip

Common Question: "How would you prevent VLAN hopping?"

Answer:

  1. Change native VLAN: "Set native VLAN to an unused VLAN (e.g., 999), not VLAN 1"
  2. Explicit trunk config: "Manually configure trunks, don't rely on DTP negotiation"
  3. Disable DTP: "Use 'switchport nonegotiate' on trunk ports"
  4. Access ports: "Set 'switchport mode access' on all end-user ports to prevent them from becoming trunks"
  5. Prune VLANs: "Only allow necessary VLANs on trunks with 'switchport trunk allowed vlan'"
