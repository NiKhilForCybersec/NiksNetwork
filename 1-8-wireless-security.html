# 1. CORPORATE (Managed devices)
SSID: Corp-Secure
Security: WPA2/WPA3-Enterprise
Auth: 802.1X (EAP-TLS with certificates)
VLAN: 10 (Corporate)
Access: Full internal + internet

# 2. BYOD (Personal devices)
SSID: Corp-BYOD  
Security: WPA2/WPA3-Enterprise
Auth: 802.1X (PEAP with AD credentials)
VLAN: 20 (BYOD - restricted)
Access: Internet + limited internal

# 3. GUEST (Visitors)
SSID: Guest
Security: WPA2-Personal or Captive Portal
Auth: Sponsored access or self-registration
VLAN: 30 (Guest - isolated)
Access: Internet only, client isolation

# 4. IOT (Devices)
SSID: IoT-Devices (hidden)
Security: WPA2-Enterprise or PSK
VLAN: 40 (IoT - heavily restricted)
Access: Specific destinations only

Wireless Security Checklist

Category Requirement
Encryption WPA2-Enterprise minimum, WPA3 preferred
Authentication 802.1X with RADIUS, EAP-TLS for high security
Segmentation Separate VLANs for Corp/BYOD/Guest/IoT
Rogue Detection WIDS/WIPS enabled on controller
Management Secure AP management (HTTPS, SSH)
Logging Send wireless logs to SIEM
PMF Enable Protected Management Frames (802.11w)
âš ï¸ Common Wireless Mistakes
  • WPA2-Personal in enterprise: Use 802.1X, not PSK
  • Weak PSK: If using PSK, 20+ random characters
  • No segmentation: Guest on same VLAN as corporate
  • Hidden SSID: Not security, just inconvenience
  • MAC filtering: Easily bypassed, not security
  • No certificate validation: Evil twin vulnerability
ğŸ’¡ Interview Tip

Question: "How would you secure an enterprise wireless network?"

Answer: "I'd implement WPA2 or WPA3-Enterprise with 802.1X authentication using a RADIUS server integrated with Active Directory. For managed devices, I'd use EAP-TLS with certificates for strongest security. I'd create separate SSIDs for corporate, BYOD, guest, and IoT - each on different VLANs with firewall rules controlling inter-VLAN traffic. Guest should only reach the internet with client isolation enabled. I'd enable WIDS/WIPS for rogue AP detection and Protected Management Frames to prevent deauth attacks. All wireless logs would go to our SIEM for monitoring. For clients, I'd configure certificate validation to prevent evil twin attacks."
