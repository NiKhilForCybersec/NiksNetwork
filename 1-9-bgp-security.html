BGP Security

ğŸ“‹ What You'll Learn

BGP (Border Gateway Protocol) is the routing protocol that holds the Internet together. It was designed without security, making it vulnerable to hijacking, leaks, and spoofing attacks that can redirect traffic globally.

  • BGP fundamentals and AS numbers
  • BGP hijacking and route leaks
  • RPKI and ROA validation
  • BGP security best practices
  • Monitoring and detection
ğŸ§  Memory Aid: BGP Security
ğŸ—ºï¸ BGP = The Internet's GPS

BGP is like the postal routing system between countries:

  • Autonomous System (AS) = A country/organization with its own postal system
  • BGP Announcement = "Hey, I can deliver to these addresses!"
  • BGP Hijacking = Fraudster saying "Send Pakistan's mail to me!" (route theft)
  • BGP Leak = Accidentally routing traffic through wrong country
ğŸ›¡ï¸ BGP Security Measures
  • RPKI/ROA = Cryptographic proof "I'm authorized to announce these routes"
  • BGP Prefix Filtering = "I'll only accept routes I expect from you"
  • AS-Path Filtering = "This route path looks suspicious, reject it"
ğŸ¯ Key Fact: BGP was built on TRUST - no built-in authentication. That's why hijacking is possible! RPKI helps prove legitimacy.

BGP Attacks

Attack Description Impact
BGP Hijacking Announce someone else's prefix as your own Traffic redirected to attacker
Prefix Hijacking Announce more specific prefix (/25 vs /24) More specific wins, traffic stolen
AS Path Manipulation Forge or modify AS path Bypass security, misdirect traffic
Route Leak Accidentally re-announce learned routes Traffic blackholed or misdirected
BGP Session Hijacking Take over BGP TCP session Inject malicious routes

BGP Security Best Practices

Control Description
MD5 Authentication Authenticate BGP sessions with shared secret
TTL Security (GTSM) Require TTL=255, prevents remote attacks
Prefix Filtering Only accept expected prefixes from peers
Max Prefix Limits Limit number of prefixes accepted
RPKI/ROV Validate route origins cryptographically
Bogon Filtering Reject private/reserved address space
AS Path Filtering Reject routes with suspicious AS paths

Question: "What is BGP hijacking and how do you prevent it?"

Answer: "BGP hijacking occurs when an AS announces routes for prefixes it doesn't own. Because BGP was designed to trust announcements, other routers accept these false routes and traffic gets misdirected to the attacker. Prevention requires multiple layers: First, create ROAs (Route Origin Authorizations) in RPKI so validators can verify your prefixes. Second, configure prefix filters on all BGP sessions to only accept expected routes. Third, use MD5 authentication and TTL security (GTSM) to protect BGP sessions themselves. Fourth, monitor your prefixes using services like BGPMon or RIPE RIS to detect unauthorized announcements. The industry is moving toward RPKI adoption - major networks now reject RPKI-invalid routes."
