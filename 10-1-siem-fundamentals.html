SIEM Fundamentals

ğŸ“‹ What You'll Learn

Security Information and Event Management (SIEM) is the cornerstone of Security Operations Centers (SOC). A SIEM collects logs from across the enterprise, correlates events, detects threats, and enables investigation and response.

  • What is SIEM and why it matters
  • SIEM architecture and components
  • Log collection and normalization
  • Correlation and detection rules
  • Alert triage and investigation

What is SIEM?

SIEM combines Security Information Management (SIM) - log collection and storage - with Security Event Management (SEM) - real-time analysis and alerting. Modern SIEMs provide centralized visibility, threat detection, and compliance reporting.

SIEM Core Functions

Function Description Example
Log Collection Aggregate logs from all sources Firewall logs, Windows events, cloud audit logs
Normalization Parse and standardize log formats Map vendor fields to common schema
Correlation Connect related events across sources Failed login + successful login = potential brute force
Alerting Notify on suspicious activity Alert when 10+ failed logins from same IP
Search Query historical data Find all connections to suspicious IP last 30 days
Dashboards Visualize security posture Top blocked threats, login anomalies
Reporting Compliance and executive reports PCI-DSS log review, monthly security summary
Retention Store logs for compliance/forensics 1 year online, 7 years archive

SIEM vs SOAR vs XDR

Platform Primary Function Key Capabilities
SIEM Log aggregation, correlation, detection Central log storage, alerts, compliance, search
SOAR Automation and orchestration Playbooks, automated response, case management
XDR Extended detection across endpoint/network/cloud Unified detection, native integration, ML-based
âœ… Best Practice - SIEM Implementation

Start small and expand:

  1. Phase 1: Core security logs (firewall, AD, VPN, email)
  2. Phase 2: Endpoint (EDR, Windows events from critical servers)
  3. Phase 3: Cloud (AWS, Azure, O365)
  4. Phase 4: Application logs, DNS, DHCP, etc.

Don't just collect - detect: Every log source should have detection rules. If you're not alerting on it, why collect it?

ğŸ’¡ Interview Tip

Common Question: "What is SIEM and how does it help with security?"

Answer:

  • Definition: SIEM (Security Information and Event Management) aggregates logs from across the environment, correlates events, and detects threats in real-time.
  • Value: Without SIEM, security teams would have to log into dozens of systems to investigate incidents. SIEM provides a single pane of glass for visibility.
  • Key capabilities: Log collection, normalization, correlation rules, alerting, investigation/search, compliance reporting, long-term retention.
  • Example: "A firewall might see suspicious traffic, AD might see failed logins, and endpoint might see process execution - but only SIEM can correlate these into a single attack story."
