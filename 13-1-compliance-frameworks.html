Compliance Frameworks

ğŸ“‹ What You'll Learn

Security compliance frameworks provide structured approaches to protecting data and systems. Understanding these frameworks is essential for enterprise security roles.

  • Major compliance frameworks overview
  • PCI-DSS requirements
  • HIPAA security rule
  • SOC 2 Trust Principles
  • NIST Cybersecurity Framework
  • ISO 27001

Framework Comparison

Framework Applies To Type Focus
PCI-DSS Payment card data handlers Mandatory Cardholder data protection
HIPAA Healthcare (US) Mandatory Protected Health Information
SOC 2 Service providers Voluntary (often required) Trust service criteria
NIST CSF Any organization Voluntary (mandatory for US Fed) Cybersecurity best practices
ISO 27001 Any organization Voluntary (certification) Information security management
GDPR EU citizen data handlers Mandatory Privacy, data protection

PCI-DSS 4.0 Requirements

Req # Requirement Key Controls
1 Install and maintain network security controls Firewalls, segmentation
2 Apply secure configurations Hardening, no defaults
3 Protect stored account data Encryption, masking, tokenization
4 Protect cardholder data during transmission TLS 1.2+, no WEP
5 Protect against malware Anti-malware, EDR
6 Develop and maintain secure systems Secure SDLC, patching
7 Restrict access by business need Least privilege, RBAC
8 Identify users and authenticate access MFA, password policies
9 Restrict physical access Badges, cameras, logs
10 Log and monitor all access SIEM, log retention
11 Test security regularly Vuln scans, pen tests
12 Support security with policies Policies, awareness training

SOC 2 Trust Service Criteria

Criteria Description Common Controls
Security Protection from unauthorized access Firewalls, IDS, access controls
Availability System accessible as agreed HA, DR, SLAs
Processing Integrity Complete, accurate processing QA, validation, monitoring
Confidentiality Protect confidential data Encryption, DLP, classification
Privacy Personal info handling Consent, retention, access rights
âœ… Best Practice - Compliance
  • Map controls: Many frameworks overlap - map once, comply many
  • Automate evidence: Use tools to generate compliance reports
  • Continuous compliance: Don't just prepare for audits
  • Document everything: Policies, procedures, exceptions
  • Risk-based approach: Focus on highest-risk areas first

Question: "What compliance frameworks have you worked with?"

Answer: "I've worked primarily with PCI-DSS and SOC 2. For PCI, I helped maintain network segmentation (Req 1), implemented logging to SIEM (Req 10), and coordinated quarterly vulnerability scans (Req 11). For SOC 2, I focused on Security and Availability criteria - implementing access controls, monitoring, and documenting our change management process. I understand that frameworks often overlap, so we mapped controls across both to avoid duplicate work."
