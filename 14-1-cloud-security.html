Common Cloud Security Risks

Risk Description Mitigation
Misconfiguration Public S3 buckets, open security groups CSPM tools, IaC scanning
Excessive Permissions Overly permissive IAM roles Least privilege, regular reviews
Exposed Credentials Keys in code, environment variables Secrets management, rotation
Data Exposure Unencrypted data, public endpoints Encryption at rest/transit, private endpoints
Lack of Logging No visibility into activity Enable CloudTrail, VPC Flow Logs
Shadow IT Unauthorized cloud usage CASB, account governance

ğŸ’¡ Interview Tip

Question: "Explain the shared responsibility model."

Answer: "The cloud provider is responsible for security OF the cloud - physical infrastructure, hypervisor, network backbone. The customer is responsible for security IN the cloud - data, IAM, application security, network controls. The split varies by service model: with IaaS like EC2, you manage the OS and everything above. With PaaS like Lambda, the provider manages more. With SaaS, you mainly manage data and access. The key is understanding what YOU are responsible for securing."
