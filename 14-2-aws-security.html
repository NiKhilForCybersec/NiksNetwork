AWS Security Services

ğŸ“‹ What You'll Learn

AWS provides a comprehensive suite of security services for threat detection, compliance, and security posture management. Understanding these services is essential for securing AWS environments.

  • GuardDuty threat detection
  • Security Hub posture management
  • Inspector vulnerability scanning
  • CloudTrail audit logging
  • IAM security best practices

AWS Security Service Overview

Service Function Key Use Case
GuardDuty Threat detection Detect malicious activity, compromised instances
Security Hub Security posture Aggregate findings, compliance checks
Inspector Vulnerability scanning EC2, ECR, Lambda vulnerability assessment
Macie Data discovery Find sensitive data in S3
CloudTrail API audit logging Who did what, when
Config Configuration tracking Resource compliance, change tracking
IAM Access Analyzer Access analysis Find unintended public/cross-account access

GuardDuty Finding Types

Category Example Findings
EC2 CryptoCurrency mining, DDoS participation, C2 communication
IAM Anomalous API calls, credential exfiltration, policy changes
S3 Unusual data access, anonymous access, suspicious IP
Kubernetes Anonymous API calls, privileged containers, suspicious exec
Malware Malware detected on EC2, malicious file upload to S3

Question: "How would you set up security monitoring in AWS?"

Answer: "I'd start with the foundational services: Enable CloudTrail in all regions for API audit logging, with logs sent to a centralized S3 bucket and CloudWatch for alerting. Enable GuardDuty for threat detection - it analyzes CloudTrail, VPC Flow Logs, and DNS logs automatically. Enable Security Hub to aggregate findings and run compliance checks against CIS and AWS best practices. For vulnerability management, enable Inspector for EC2 and container scanning. Finally, set up EventBridge rules to send critical findings to our SIEM or create tickets automatically. All of these should be enabled organization-wide using AWS Organizations."
