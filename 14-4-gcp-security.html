    --direction=INGRESS --priority=100 --action=DENY \
    --rules=all --source-ranges=1.2.3.4/32

    --direction=INGRESS --priority=1000 --action=ALLOW \
    --rules=tcp:22 --source-ranges=10.0.0.0/8 --target-tags=allow-ssh

Question: "How does GCP's security model differ from AWS and Azure?"

Answer: "GCP's security is built around organization hierarchy and resource inheritance. IAM policies cascade down from Organization â†’ Folders â†’ Projects â†’ Resources, which is more flexible than AWS's flat account model. GCP has VPC Service Controls for data exfiltration prevention - a unique feature for creating security perimeters around sensitive APIs.
