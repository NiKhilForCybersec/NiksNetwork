  --priority 100 \
  --direction Inbound \
  --access Allow \
  --protocol Tcp \
  --source-address-prefixes Internet \

az network nsg rule create \
  --resource-group MyRG \
  --nsg-name web-nsg \
  --name Deny-All-Inbound \
  --priority 4096 \
  --direction Inbound \
  --access Deny \
  --protocol '*' \
  --source-address-prefixes '*' \
  --destination-port-ranges '*'

Azure Network Security Best Practices

Practice Description
Hub-spoke topology Centralize security controls in hub VNet
Azure Firewall in hub Inspect all traffic through central firewall
NSG per subnet Defense in depth at subnet level
Use ASGs Logical grouping instead of IP-based rules
Enable flow logs NSG flow logs + Traffic Analytics
Private Link Private access to PaaS services
Azure Bastion Secure RDP/SSH without public IPs
DDoS Protection Enable DDoS Protection Standard
ğŸ’¡ Interview Tip

Question: "How would you design a secure Azure network architecture?"

Answer: "I'd implement a hub-spoke architecture. The hub VNet contains shared services: Azure Firewall for traffic inspection, VPN/ExpressRoute gateway for hybrid connectivity, and Azure Bastion for secure admin access. Spoke VNets are peered to the hub and contain workloads separated by environment (prod, dev) or application. User-defined routes force all traffic through Azure Firewall. Each subnet has an NSG with explicit allow rules using Application Security Groups for logical grouping. I'd enable NSG flow logs with Traffic Analytics for visibility. For PaaS services, I'd use Private Link to keep traffic private. Azure DDoS Protection Standard protects public endpoints. This gives defense in depth: Azure Firewall for L3-L7 inspection, NSGs for subnet-level filtering, and full logging for security monitoring."
