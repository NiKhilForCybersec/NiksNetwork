# Limit new connections per IP
iptables -A INPUT -p tcp --syn -m connlimit --connlimit-above 50 -j DROP

# Rate limit ICMP
iptables -A INPUT -p icmp --icmp-type echo-request -m limit --limit 1/s -j ACCEPT
iptables -A INPUT -p icmp --icmp-type echo-request -j DROP

# Block invalid packets
iptables -A INPUT -m state --state INVALID -j DROP

Phase Actions
1. Detection Monitor alerts, identify attack type and vector
2. Classification Volumetric? Protocol? Application? Size?
3. Mitigation Engage appropriate protection layer
4. Communication Notify stakeholders, update status page
5. Monitoring Watch for attack evolution, tune filters
6. Post-Incident Document, analyze, improve defenses

ğŸ’¡ Interview Tip

Question: "How would you protect against a DDoS attack?"

Answer: "I'd implement defense in depth. For volumetric attacks, use cloud scrubbing services like Cloudflare or AWS Shield that have massive bandwidth to absorb floods - these use anycast to distribute traffic globally. For protocol attacks like SYN floods, enable SYN cookies on servers and use stateful firewalls with connection limits. For application layer attacks, deploy a WAF with rate limiting and bot detection. Have BGP blackhole (RTBH) capability coordinated with your ISP for emergency traffic dropping. Most importantly, know your normal traffic baseline so you can detect anomalies quickly. Have runbooks ready with escalation contacts for your ISP and scrubbing provider. Test your defenses regularly."
