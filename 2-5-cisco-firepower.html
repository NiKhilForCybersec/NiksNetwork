Cisco Firepower and ASA

ğŸ“‹ What You'll Learn

Cisco offers two major firewall platforms: the legacy ASA (Adaptive Security Appliance) and the next-generation Firepower Threat Defense (FTD). Understanding both is essential as many organizations still run ASA or are migrating to FTD.

  • ASA vs Firepower (FTD) - key differences
  • Firepower Management Center (FMC)
  • Access Control Policies
  • Intrusion Prevention (Snort)
  • AnyConnect VPN configuration
  • Essential CLI commands
  • Migration strategies from ASA to FTD
ğŸ§  Memory Aid: Cisco Firepower
ğŸ”§ Cisco Firewall Evolution

Understanding Cisco's firewall journey:

  • PIX = The grandfather - first Cisco firewall, retired
  • ASA = The parent - stateful firewall, still widely used
  • Firepower (FTD) = The modern child - ASA + advanced threat protection (IPS, AMP, URL filtering)
ğŸ“‹ Key Cisco Terms
  • FMC = Firepower Management Center - the brain that manages all FTD devices
  • FTD = Firepower Threat Defense - the actual firewall device/software
  • ACP = Access Control Policy - contains all your firewall rules
  • Snort = The IPS engine inside Firepower
ğŸ¯ Remember: "FMC manages FTD. Policies are deployed from FMC to FTD. Always stage changes before deploying!"

Migration: ASA to FTD

1

Assessment

Analyze current ASA configuration. Use Cisco Firewall Migration Tool to identify compatibility issues. Document all ACLs, NAT rules, VPN configs.

2

Prepare FMC

Deploy FMC (appliance or VM). Import ASA configuration using migration tool. This creates equivalent FTD objects and policies.

3

Validate & Enhance

Review migrated policies. Add NGFW features: application control, IPS policies, URL filtering. Test in lab if possible.

4

Reimage or Replace

For supported hardware, reimage from ASA to FTD. For older hardware, replace with new FTD-capable appliance. Register to FMC.

5

Deploy & Monitor

Push policies from FMC. Monitor closely for issues. Start with IPS in detection mode, then enable blocking after tuning.

  • Not all features migrate 1:1: Some ASA features work differently or don't exist in FTD

  • Performance: IPS inspection adds overhead - size accordingly
  • Management change: Teams used to CLI must adapt to FMC GUI workflow

Common Question: "What's the difference between ASA and FTD?"

Answer:

  • ASA: Traditional stateful firewall (L3/L4). Uses ACLs based on IP/port. No built-in application awareness or IPS. Managed via CLI or ASDM GUI. Still widely deployed for VPN.
  • FTD: Next-gen firewall combining ASA's stateful engine with Firepower's NGFW capabilities. Has application visibility, integrated IPS (Snort), URL filtering, malware protection. Managed via FMC for full features.

Key point: "FTD is essentially ASA + FirePOWER combined into a unified image. If you know ASA, the underlying connection handling is similar, but policy management and NGFW features are completely different."
