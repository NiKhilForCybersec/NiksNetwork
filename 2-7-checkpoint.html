Secure Internal Communication (SIC)

SIC is Check Point's certificate-based authentication and encryption for communication between all Check Point components.

âœ… Best Practice - SIC Troubleshooting

SIC issues are common during deployment. Check these:

  • One-time password: Must match between SMS object and gateway initialization
  • Time sync: NTP must be configured, time difference causes SIC failure
  • DNS resolution: Gateway must resolve SMS hostname
  • Network connectivity: Port 18191 must be open between SMS and gateway

Reset SIC on gateway: cpconfig â†’ Reset SIC â†’ Reinitialize from SmartConsole

Common Check Point operations:

  • Policy changes: Create/modify rules in SmartConsole, publish, install policy
  • Log analysis: Search SmartLog for traffic, investigate blocks
  • HA management: Monitor cluster state, perform failovers
  • VPN support: Troubleshoot site-to-site and remote access VPN
  • Hotfix installation: Apply hotfixes during maintenance windows
  • Backup: Regular policy and configuration backups

Common Question: "Explain Check Point's three-tier architecture."

Answer:

  • SmartConsole: The GUI client installed on admin workstations. Used for policy creation, log viewing, and monitoring.
  • Security Management Server (SMS): Central server that stores all policies and objects. Compiles policies and pushes them to gateways. Can also collect logs.
  • Security Gateway: The enforcement point. Receives compiled policy from SMS and enforces it on traffic. This is where actual inspection happens.

Key benefit: "This separation allows centralized management of multiple gateways, role-based access, and better scalability. Changes made in SmartConsole are stored on SMS and only take effect when policy is installed to gateways."
