AWS Security Groups & NACLs

Feature Security Group Network ACL (NACL)
Level Instance (ENI) Subnet
State Stateful (return traffic auto-allowed) Stateless (must allow both directions)
Rules Allow only (implicit deny) Allow and Deny
Evaluation All rules evaluated Rules processed in order (first match)
Default Deny all inbound, Allow all outbound Allow all (default NACL)
Use Case Primary firewall for instances Additional subnet-level protection

ğŸ’¡ Interview Tip

Common Question: "What's the difference between AWS Security Groups and NACLs?"

Answer:

  • Security Groups: Instance-level, stateful (return traffic auto-allowed), allow rules only, all rules evaluated
  • NACLs: Subnet-level, stateless (must explicitly allow return traffic), can have allow AND deny rules, processed in order by rule number
  • Best practice: "Use Security Groups as primary firewall for micro-segmentation. Use NACLs as an additional layer for subnet-wide blocks, like blocking known bad IP ranges or enforcing subnet isolation."
