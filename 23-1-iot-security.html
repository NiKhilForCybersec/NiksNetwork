IoT & OT Security

ğŸ“‹ What You'll Learn

IoT (Internet of Things) and OT (Operational Technology) devices present unique security challenges - they often can't be patched, run legacy software, and expand the attack surface significantly.

  • IoT/OT security challenges
  • Network segmentation
  • Device discovery and inventory
  • IoT security platforms
  • ICS/SCADA security
ğŸ§  Memory Aid: IoT Security
ğŸ  IoT = Smart Home Gone Wild

IoT security challenges:

  • Default Credentials = Front door with "password" as the key
  • No Updates = Security holes never get patched
  • Weak Encryption = Conversations anyone can hear
  • Botnet Risk = Devices recruited into attack armies (Mirai)
ğŸ›¡ï¸ IoT Security Strategy
  • Segment = Put IoT on separate VLAN - don't mix with corporate
  • Inventory = Know what's on your network
  • Default Creds = Change them immediately!
  • Network Monitoring = Watch for anomalous behavior
  • Disable Unused = Turn off unnecessary features/ports
ğŸ¯ IoT Mantra: "Assume it's insecure, segment it, monitor it, update it if possible, replace it if not"

ğŸ’¡ Interview Tip

Question: "How would you secure IoT devices in an enterprise?"

Answer: "IoT security starts with visibility - you can't secure what you don't know about. I'd deploy an IoT security platform like Armis or Forescout to discover all devices. Then network segmentation - IoT devices go on isolated VLANs with strict firewall rules allowing only necessary communication. For example, cameras can only talk to the NVR, printers only accept print jobs. I'd change default credentials, disable unnecessary services, and establish a firmware update process. NAC enforces segmentation by auto-assigning devices to correct VLANs. Finally, monitor IoT traffic for anomalies - if a camera starts making DNS queries to unusual domains, that's suspicious. The key principle is zero trust - assume IoT devices are compromised and limit what they can access."
