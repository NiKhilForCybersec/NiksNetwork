ğŸ’¡ Interview Tip

Question: "How would you secure an OT/ICS network?"

Answer: "OT security differs from IT because availability is paramount - you can't just patch or reboot systems controlling physical processes. I'd implement the Purdue Model for network segmentation, with strict firewall rules between levels. An industrial DMZ separates IT from OT, ideally with data diodes for one-way data flow to historians. Remote access goes through jump servers with MFA, never direct VPN to OT. For monitoring, I'd deploy passive OT-aware solutions like Claroty or Dragos that understand ICS protocols and can detect anomalies without active scanning, which can crash PLCs. No vulnerability scanning on OT networks - instead, passive asset inventory. Critical safety systems (SIS) get their own isolated network. Default credentials must be changed, but patching requires extensive testing in lab environments first. The key principle is defense in depth while prioritizing operational availability."
