VPN Fundamentals

ğŸ“‹ What You'll Learn

A Virtual Private Network (VPN) creates a secure, encrypted tunnel over an untrusted network (typically the internet). VPNs are fundamental to enterprise networking, enabling secure connectivity between sites and for remote workers.

  • What is a VPN and why it matters
  • Site-to-Site vs Remote Access VPN

What is a VPN?

A VPN extends a private network across a public network, enabling users to send and receive data as if their devices were directly connected to the private network. The connection is encrypted, providing confidentiality and integrity.

VPN Protocol Comparison

Protocol Type Encryption Ports Best For
IPSec L3 (Network) AES, 3DES UDP 500, 4500, ESP (50) Site-to-site, enterprise
SSL/TLS VPN L4-L7 TLS 1.2/1.3 TCP 443 Remote access, restrictive networks
WireGuard L3 (Network) ChaCha20, Curve25519 UDP 51820 Modern, high-performance
OpenVPN L2/L3 OpenSSL (AES) UDP/TCP 1194 (or any) Flexible, open source
IKEv2 L3 (with IPSec) AES UDP 500, 4500 Mobile devices, fast reconnect
L2TP/IPSec L2 over IPSec IPSec (AES) UDP 500, 4500, 1701 Legacy, built-in OS support
âš ï¸ Deprecated Protocols - Avoid
  • PPTP: Broken encryption, easily cracked. Never use.
  • L2TP without IPSec: No encryption. Always pair with IPSec.
  • IPSec with 3DES: Deprecated. Use AES-256 instead.
  • IPSec with MD5/SHA-1: Weak hashing. Use SHA-256 or higher.

âœ… Best Practice - Split Tunnel Security

If using split tunnel, implement compensating controls:

  • Deploy endpoint protection (EDR) on all devices
  • Use DNS filtering (block malicious domains)
  • Implement SASE/cloud security for internet traffic
  • Require device compliance before VPN connection
  • Consider split tunnel only for trusted, managed devices

ğŸ’¡ Interview Tip

Common Question: "What's the difference between site-to-site and remote access VPN?"

Answer:

  • Site-to-Site: Connects two networks together (e.g., HQ to branch). Tunnel is between network devices (firewalls). Always-on, transparent to users. Uses IPSec typically.
  • Remote Access: Connects individual users to the network. Tunnel is from user's device to VPN gateway. User initiates connection. Uses SSL VPN or IPSec with client software.

When to use: "Site-to-site for connecting offices or data centers. Remote access for employees working from home or traveling."
