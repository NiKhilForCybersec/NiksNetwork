SSL VPN

ğŸ“‹ What You'll Learn

SSL VPN uses TLS (formerly SSL) to provide secure remote access. It's often preferred over IPSec for remote workers because it works through firewalls and NAT without special configuration.

  • SSL VPN vs IPSec VPN
  • SSL VPN modes (portal vs tunnel)
  • Split tunneling
  • Client configuration
  • Security considerations

SSL VPN vs IPSec VPN

Feature SSL VPN IPSec VPN
Protocol Layer Layer 4-7 (TLS) Layer 3 (IP)
Port TCP 443 (HTTPS) UDP 500, 4500, ESP
Firewall Traversal Excellent (looks like HTTPS) May require NAT-T, port forwarding
Client Required Browser (portal) or client Always requires client
Best For Remote access, BYOD Site-to-site, full network access
Granular Control Can limit to specific apps Full network access typically

  • CVE vulnerabilities: SSL VPN appliances are high-value targets (Pulse Secure, Fortinet CVEs)
  • Credential theft: VPN credentials are prime phishing targets
  • Split tunneling: Can expose corporate network to infected home devices
  • Patch quickly: VPN appliances must be patched immediately
âœ… Best Practice - SSL VPN Security
  • Enable MFA: Require multi-factor authentication for all VPN access
  • Use certificates: Machine certificates + user credentials
  • Host checking: Verify endpoint health before granting access
  • Limit access: Least privilege - only what users need
  • Monitor: Log all VPN connections, alert on anomalies
  • Patch rapidly: VPN vulnerabilities are actively exploited

Question: "What's the difference between split and full tunneling?"

Answer: "Full tunneling sends ALL traffic through the VPN - more secure because corporate security controls inspect everything, but higher latency and bandwidth usage. Split tunneling only sends corporate traffic through VPN while internet traffic goes direct - better performance but internet traffic bypasses security. For high-security environments, I'd recommend full tunnel. For general use, split tunnel with a cloud proxy (like Zscaler) provides a good balance."
