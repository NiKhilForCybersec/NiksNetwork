Question: "What are the security considerations when deploying SD-WAN?"

Answer: "SD-WAN introduces direct internet access at branches, which means security can't just live at the data center anymore. Key considerations: First, all overlay traffic must be encrypted with strong IPSec - AES-256 with PFS. Second, segment traffic into different VRFs so a breach in one segment doesn't affect others. Third, since branches now have direct internet access, deploy local security - either integrated NGFW in the SD-WAN appliance or cloud security via SASE. Fourth, secure the controller - use MFA, RBAC, and ensure it's highly available since it controls all policy. Fifth, implement zero trust principles - authenticate and authorize every connection. Many organizations now combine SD-WAN with SASE to get cloud-delivered security for all users and branches consistently."
