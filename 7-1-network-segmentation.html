Network Segmentation

ğŸ“‹ What You'll Learn

Network segmentation divides a network into smaller segments to contain threats, limit lateral movement, and enforce security policies. It's a fundamental security control for defense in depth.

  • Why segmentation matters
  • Segmentation approaches
  • Common network zones
  • Microsegmentation
  • Implementation strategies
ğŸ§  Memory Aid: Network Segmentation
ğŸš¢ The Ship Compartments Analogy

Network segmentation is like a ship's watertight compartments:

  • No Segmentation = Open hull - one hole sinks the whole ship!
  • Segmented Network = Watertight compartments - one breach is contained, ship stays afloat
  • Firewall Between Segments = Controlled doors between compartments - only authorized crew can pass
  • Micro-segmentation = Each room is its own compartment - maximum containment
ğŸ›ï¸ Common Segments (Zones)
  • DMZ = Visitor lobby - public-facing services, don't trust completely
  • Internal/Trust = Employee offices - regular business operations
  • Restricted = Executive floor - sensitive systems, extra protection
  • Guest = Visitor WiFi - internet only, no internal access
  • Management = Server room - admin access only, heavily protected
ğŸ¯ Key Principle: "Segment by sensitivity and function, control traffic between segments, assume breach in any segment"

âœ… Best Practice - Segmentation
  • Start with data classification: Know what you're protecting
  • Map traffic flows: Understand legitimate communication patterns
  • Default deny: Block all, allow only required traffic
  • Segment by risk: High-value assets in separate segments
  • Monitor east-west: Log traffic between segments
  • Limit management access: Jump boxes, PAM for admin access
ğŸ’¡ Interview Tip

Question: "How would you design network segmentation for an enterprise?"

Answer: "I'd start by identifying critical assets and data classification. Then create zones based on function and trust level: DMZ for public-facing services, user zone for workstations, application zone for internal apps, and a highly restricted data zone for databases. VLANs provide Layer 2 separation, firewalls enforce policies between zones. For the data center, I'd add microsegmentation to control east-west traffic between workloads. Management network should be isolated with jump boxes. The goal is limiting blast radius - if an attacker compromises one segment, they can't easily move laterally."
