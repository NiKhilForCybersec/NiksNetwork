Method How It Works Use Case

MAB MAC Authentication Bypass Devices without 802.1X (printers, IoT)
Web Auth Captive portal login Guest access, BYOD
Profiling Identify device type automatically IoT, unknown devices

Posture Assessment

Check What It Verifies Action if Failed
Antivirus AV installed, running, updated Quarantine + remediation link
Patch Level OS/app patches within policy Limited access + WSUS redirect
Encryption Disk encryption enabled Block sensitive resources
EDR Agent Required security agent present Install agent before full access
Firewall Host firewall enabled Enable before proceeding
Domain Joined Device in AD domain BYOD VLAN if not joined

âœ… NAC Implementation Best Practices
  • Start in monitor mode: Learn traffic patterns before enforcing
  • Profile first: Identify all devices before writing policies
  • Phased rollout: One building/VLAN at a time
  • Have exceptions process: Not everything supports 802.1X
  • Communicate to users: They'll call helpdesk when blocked
  • Integrate with other tools: EDR, SIEM, vulnerability scanners
ğŸ’¡ Interview Tip

Question: "How does NAC improve network security?"

Answer: "NAC ensures only authorized and compliant devices connect to the network. It works by first identifying the device through 802.1X authentication or profiling, then checking posture - is the AV running? Are patches current? Based on this, NAC assigns the device to an appropriate VLAN with matching access policies. Non-compliant devices go to a remediation VLAN where they can only access update servers. Guest devices get internet-only access. This prevents compromised or unmanaged devices from accessing sensitive resources. It also provides visibility into all connected devices, which is crucial for IoT security. The key is the combination of identity verification, health assessment, and dynamic access control."
