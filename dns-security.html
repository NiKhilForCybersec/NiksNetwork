DNSSEC (DNS Security Extensions)

DNSSEC adds cryptographic signatures to DNS records, allowing resolvers to verify that responses are authentic and haven't been tampered with.

DNSSEC Chain of Trust 
┌─────────────────────────────────────────────────────────────────────────────────────┐
│                           DNSSEC CHAIN OF TRUST                                     │
├─────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                     │
│   ROOT ZONE (.)                                                                     │
│   ┌─────────────────────────────────────────────────────────────────────────────┐  │
│   │  KSK (Key Signing Key) - Signs the ZSK                                      │  │
│   │  ZSK (Zone Signing Key) - Signs the records                                 │  │
│   │  DS record points to .com KSK                                               │  │
│   └───────────────────────────────────┬─────────────────────────────────────────┘  │
│                                       │ DS record                                  │
│                                       ▼                                            │
│   .COM TLD                                                                          │
│   ┌─────────────────────────────────────────────────────────────────────────────┐  │
│   │  KSK - verified by DS record from root                                      │  │
│   │  ZSK - Signs .com zone records                                              │  │
│   │  DS record points to example.com KSK                                        │  │
│   └───────────────────────────────────┬─────────────────────────────────────────┘  │
│                                       │ DS record                                  │
│                                       ▼                                            │
│   EXAMPLE.COM                                                                       │
│   ┌─────────────────────────────────────────────────────────────────────────────┐  │
│   │  KSK - verified by DS record from .com                                      │  │
│   │  ZSK - Signs example.com records                                            │  │
│   │                                                                             │  │
│   │  A record: www.example.com → 93.184.216.34                                  │  │
│   │  RRSIG: Signature proving A record is authentic                             │  │
│   └─────────────────────────────────────────────────────────────────────────────┘  │
│                                                                                     │
│                                                                                     │
│   DNSSEC RECORD TYPES:                                                              │
│   ────────────────────                                                              │
│   • DNSKEY: Public keys (KSK and ZSK)                                              │
│   • DS: Delegation Signer (hash of child's KSK, stored in parent)                  │
│   • RRSIG: Signature for each record set                                           │
│   • NSEC/NSEC3: Proves non-existence of records                                    │
│                                                                                     │
│                                                                                     │
│   VALIDATION PROCESS:                                                               │
│   ───────────────────                                                               │
│   1. Resolver queries for www.example.com                                          │
│   2. Gets A record + RRSIG                                                         │
│   3. Gets DNSKEY from example.com                                                  │
│   4. Verifies RRSIG with ZSK                                                       │
│   5. Verifies ZSK with KSK                                                         │
│   6. Gets DS from .com, verifies KSK hash                                          │
│   7. Continues up to root (trust anchor)                                           │
│                                                                                     │
└─────────────────────────────────────────────────────────────────────────────────────┘

DNSSEC Configuration (BIND)

Config BIND DNSSEC Configuration 
# =============================================================================
# DNSSEC SIGNING WITH BIND
# =============================================================================

# Generate ZSK (Zone Signing Key)
dnssec-keygen -a ECDSAP256SHA256 -n ZONE example.com

# Generate KSK (Key Signing Key)
dnssec-keygen -a ECDSAP256SHA256 -n ZONE -f KSK example.com

# Sign the zone
dnssec-signzone -A -3 $(head -c 1000 /dev/urandom | sha1sum | cut -b 1-16) \
    -N INCREMENT -o example.com -t db.example.com

# BIND named.conf configuration
zone "example.com" {
    type master;
    file "/var/named/db.example.com.signed";
    key-directory "/var/named/keys";
    auto-dnssec maintain;
    inline-signing yes;
    dnssec-policy default;
};

# Enable DNSSEC validation on resolver
options {
    dnssec-validation auto;
    # Or explicitly set trust anchor
    # dnssec-validation yes;
};

# Verify DNSSEC
dig +dnssec example.com
dig +sigchase example.com
delv @8.8.8.8 example.com

DNS over HTTPS (DoH) & DNS over TLS (DoT)

Feature Traditional DNS DoT (DNS over TLS) DoH (DNS over HTTPS)
Port UDP/53 TCP/853 TCP/443
Encryption None TLS HTTPS
Blockable Easy Easy (port 853) Hard (looks like HTTPS)
Enterprise Control Easy Easy Difficult
Privacy None ISP can't see queries ISP can't see queries

DoT/DoH Server Configuration

Config Unbound DoT/DoH Configuration 
# =============================================================================
# UNBOUND DNS OVER TLS SERVER
# =============================================================================

# /etc/unbound/unbound.conf
server:
    interface: 0.0.0.0
    port: 53
    
    # DNS over TLS
    interface: 0.0.0.0@853
    tls-service-key: "/etc/unbound/server.key"
    tls-service-pem: "/etc/unbound/server.pem"
    tls-port: 853
    
    # Security settings
    hide-identity: yes
    hide-version: yes
    
    # Access control
    access-control: 10.0.0.0/8 allow
    access-control: 192.168.0.0/16 allow
    
    # DNSSEC validation
    auto-trust-anchor-file: "/var/lib/unbound/root.key"
    
    # Hardening
    harden-glue: yes
    harden-dnssec-stripped: yes
    harden-referral-path: yes
    
    # Privacy
    qname-minimisation: yes
    aggressive-nsec: yes

# Forward to upstream DoT servers
forward-zone:
    name: "."
    forward-tls-upstream: yes
    # Cloudflare
    forward-addr: 1.1.1.1@853#cloudflare-dns.com
    forward-addr: 1.0.0.1@853#cloudflare-dns.com
    # Google
    forward-addr: 8.8.8.8@853#dns.google
    forward-addr: 8.8.4.4@853#dns.google

DNS Filtering & Response Policy Zones

Config BIND Response Policy Zone (RPZ) 
# =============================================================================
# RESPONSE POLICY ZONES (RPZ) - DNS FILTERING
# =============================================================================

# /etc/bind/named.conf.options
options {
    response-policy {
        zone "rpz.local";
        zone "malware-block.rpz" policy nxdomain;
    };
};

# /etc/bind/named.conf.local
zone "rpz.local" {
    type master;
    file "/etc/bind/zones/db.rpz.local";
    allow-query { none; };
};

# /etc/bind/zones/db.rpz.local
$TTL 1H
@    SOA localhost. root.localhost. (
         1    ; Serial
         1H   ; Refresh
         15M  ; Retry
         1W   ; Expire
         1D ) ; Minimum TTL

     NS  localhost.

; Block specific domain
malware.com         CNAME   .    ; NXDOMAIN
*.malware.com       CNAME   .    ; Block all subdomains

; Redirect to walled garden
blocked.com         A       10.0.0.100
*.blocked.com       A       10.0.0.100

; Block based on IP (if DNS returns this IP, block)
32.100.0.0.10.rpz-ip    CNAME   .

; Block by nameserver
ns1.badnameserver.com.rpz-nsdname    CNAME   .

; Passthrough (whitelist)
allowed.com         CNAME   rpz-passthru.

Pi-hole DNS Sinkhole

Pi-hole is a network-wide ad blocker that acts as a DNS sinkhole, blocking ads and tracking at the DNS level for all devices on your network.

Bash Pi-hole Installation & Configuration 
# =============================================================================
# PI-HOLE INSTALLATION
# =============================================================================

# One-line install (review script first!)
curl -sSL https://install.pi-hole.net | bash

# Or manual install
git clone --depth 1 https://github.com/pi-hole/pi-hole.git Pi-hole
cd "Pi-hole/automated install/"
sudo bash basic-install.sh

# Set password for web interface
pihole -a -p

# =============================================================================
# PI-HOLE CONFIGURATION
# =============================================================================

# /etc/pihole/pihole-FTL.conf
PRIVACYLEVEL=0           # Show everything
MAXDBDAYS=365            # Keep 1 year of data
DBINTERVAL=1.0           # Update DB every minute
MAXLOGAGE=24.0           # Log queries for 24 hours

# /etc/pihole/setupVars.conf
PIHOLE_INTERFACE=eth0
IPV4_ADDRESS=192.168.1.10/24
PIHOLE_DNS_1=1.1.1.1
PIHOLE_DNS_2=1.0.0.1
QUERY_LOGGING=true
INSTALL_WEB_SERVER=true
INSTALL_WEB_INTERFACE=true
LIGHTTPD_ENABLED=true
DNSSEC=true
REV_SERVER=true
REV_SERVER_CIDR=192.168.1.0/24
REV_SERVER_TARGET=192.168.1.1
REV_SERVER_DOMAIN=local

# =============================================================================
# PIHOLE CLI COMMANDS
# =============================================================================

# Status
pihole status

# Update gravity (blocklists)
pihole -g

# Update Pi-hole
pihole -up

# Query log
pihole -t                # Tail log
pihole -q example.com    # Query specific domain

# Whitelist/Blacklist
pihole -w example.com    # Whitelist
pihole -b malware.com    # Blacklist
pihole --wild malware.com # Wildcard blacklist

# Disable/Enable
pihole disable 5m        # Disable for 5 minutes
pihole enable

# Statistics
pihole -c                # Chronometer (stats)

# =============================================================================
# RECOMMENDED BLOCKLISTS
# =============================================================================

# Add to /etc/pihole/adlists.list or via web interface

# Default Pi-hole lists
https://raw.githubusercontent.com/StevenBlack/hosts/master/hosts

# Malware domains
https://urlhaus.abuse.ch/downloads/rpz/

# Phishing
https://phishing.army/download/phishing_army_blocklist.txt

DNS Server Hardening

Config BIND Hardening 
# =============================================================================
# BIND DNS SERVER HARDENING
# =============================================================================

options {
    # Run in chroot
    directory "/var/named";
    
    # Hide version
    version "not disclosed";
    hostname "not disclosed";
    
    # Disable recursion on authoritative servers
    recursion no;               # For authoritative only
    # recursion yes;            # For resolvers
    # allow-recursion { internal; };
    
    # Disable zone transfers except to secondaries
    allow-transfer { none; };
    # allow-transfer { 192.168.1.2; };  # Secondary NS only
    
    # Limit query sources
    allow-query { any; };       # For authoritative
    # allow-query { internal; };  # For internal resolver
    
    # Rate limiting (RRL) - prevent amplification attacks
    rate-limit {
        responses-per-second 10;
        window 5;
        slip 2;
        ipv4-prefix-length 24;
        ipv6-prefix-length 56;
    };
    
    # DNSSEC
    dnssec-validation auto;
    
    # Disable unused features
    allow-update { none; };
    allow-notify { none; };
    
    # Logging
    querylog yes;
    
    # Randomize source port and query ID
    use-v4-udp-ports { range 10000 60000; };
    use-v6-udp-ports { range 10000 60000; };
    
    # Cookie support (DNS cookies for spoofing prevention)
    send-cookie yes;
    
    # Minimize responses
    minimal-responses yes;
    
    # Fetch glue even if not in-bailiwick
    fetch-glue no;
};

# Access control lists
acl "internal" {
    10.0.0.0/8;
    192.168.0.0/16;
    172.16.0.0/12;
    localhost;
};

acl "secondaries" {
    192.168.1.2;
    192.168.1.3;
};

# Logging configuration
logging {
    channel security_log {
        file "/var/log/named/security.log" versions 5 size 10m;
        severity info;
        print-time yes;
        print-severity yes;
        print-category yes;
    };
    
    channel query_log {
        file "/var/log/named/query.log" versions 5 size 100m;
        severity info;
        print-time yes;
    };
    
    category security { security_log; };
    category queries { query_log; };
};

DNS Monitoring & Threat Detection

Bash DNS Monitoring Commands 
# =============================================================================
# DNS MONITORING & ANALYSIS
# =============================================================================

# Real-time query logging
tcpdump -i eth0 -n port 53

# Detailed DNS capture
tcpdump -i eth0 -n -s0 port 53 -w dns_capture.pcap

# Parse BIND query log
tail -f /var/log/named/query.log

# Look for DNS tunneling indicators
# - Unusually long domain names
# - High query volume to single domain
# - TXT record queries
# - Encoded/random subdomain names

grep -E "query.{50,}" /var/log/named/query.log
awk '{print $4}' /var/log/named/query.log | sort | uniq -c | sort -rn | head

# DNS statistics (BIND)
rndc stats
cat /var/named/data/named_stats.txt

# =============================================================================
# DETECT DNS TUNNELING
# =============================================================================

# Look for long domain names (possible tunneling)
cat /var/log/named/query.log | \
  awk -F'(' '{print $2}' | \
  awk -F')' '{print length($1), $1}' | \
  sort -rn | head -20

# High entropy domain detection
# Tunneled data often has random-looking subdomains
# Example malicious: a8f3b2c1d4e5.evil.com

# Find domains with many TXT queries (common for tunneling)
grep "TXT" /var/log/named/query.log | \
  awk '{print $4}' | sort | uniq -c | sort -rn | head

# =============================================================================
# SPLUNK/SIEM QUERIES FOR DNS
# =============================================================================

# High volume DNS to single domain (possible C2)
# index=dns | stats count by query | where count > 1000

# Long domain names (tunneling)
# index=dns | eval length=len(query) | where length > 50

# TXT record queries
# index=dns query_type=TXT | stats count by query

# DNS to non-standard ports
# index=network dest_port!=53 app=dns

# NXDOMAIN flood (DGA malware)
# index=dns reply_code=NXDOMAIN | stats count by src_ip | where count > 100
