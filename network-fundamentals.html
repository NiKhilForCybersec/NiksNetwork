Network Fundamentals

Master the essential networking concepts that form the foundation of all network security. From the OSI model to routing protocols, this comprehensive guide covers everything you need.

OSI Model (Open Systems Interconnection)

The OSI model is a conceptual framework that standardizes network communication into seven distinct layers. Each layer has specific functions and communicates with adjacent layers.

TCP/IP Model

IPv4 Address Structure

Subnetting Example

Example Subnet Calculation: 192.168.10.0/26 
Network: 192.168.10.0/26
Subnet Mask: 255.255.255.192

Calculation:
- /26 means 26 network bits, 6 host bits
- 2^6 = 64 addresses per subnet
- 64 - 2 = 62 usable hosts (minus network and broadcast)

Subnet Breakdown:
┌─────────────────────────────────────────────────────────────────────────────┐
│ Subnet │    Network     │   First Host   │   Last Host    │   Broadcast   │
├────────┼────────────────┼────────────────┼────────────────┼───────────────┤
│   1    │ 192.168.10.0   │ 192.168.10.1   │ 192.168.10.62  │ 192.168.10.63 │
│   2    │ 192.168.10.64  │ 192.168.10.65  │ 192.168.10.126 │ 192.168.10.127│
│   3    │ 192.168.10.128 │ 192.168.10.129 │ 192.168.10.190 │ 192.168.10.191│
│   4    │ 192.168.10.192 │ 192.168.10.193 │ 192.168.10.254 │ 192.168.10.255│
└─────────────────────────────────────────────────────────────────────────────┘

Quick Formula:
- Number of subnets = 2^(borrowed bits)
- Hosts per subnet = 2^(host bits) - 2
- Block size = 256 - subnet mask octet value

TCP vs UDP

TCP vs UDP Comparison 
TCP (Transmission Control Protocol)            UDP (User Datagram Protocol)
═══════════════════════════════════            ═══════════════════════════════

  ┌────────────┐       ┌────────────┐            ┌────────────┐       ┌────────────┐
  │   Client   │       │   Server   │            │   Client   │       │   Server   │
  └─────┬──────┘       └─────┬──────┘            └─────┬──────┘       └─────┬──────┘
        │                    │                         │                    │
        │──── SYN ──────────►│                         │──── Data ─────────►│
        │                    │                         │                    │
        │◄─── SYN-ACK ───────│                         │◄─── Data ──────────│
        │                    │                         │                    │
        │──── ACK ──────────►│                         │──── Data ─────────►│
        │                    │                         │                    │
        │◄──── Data ────────►│                         └────────────────────┘
        │    (Reliable)      │
        │                    │                   • No handshake
        │──── FIN ──────────►│                   • No acknowledgments
        │◄─── ACK ───────────│                   • No ordering guarantee
        │◄─── FIN ───────────│                   • Faster, lower overhead
        │──── ACK ──────────►│                   • "Fire and forget"
        └────────────────────┘

  • Connection-oriented                        • Connectionless
  • Reliable delivery                          • Best-effort delivery
  • Flow control                               • No flow control
  • Ordered delivery                           • No ordering
  • Error checking + recovery                  • Error checking only
Feature TCP UDP
Connection Connection-oriented (3-way handshake) Connectionless
Reliability Guaranteed delivery, retransmission No guarantee, no retransmission
Ordering Packets delivered in order No ordering guarantee
Speed Slower (overhead) Faster (minimal overhead)
Header Size 20-60 bytes 8 bytes
Flow Control Yes (sliding window) No
Use Cases HTTP, HTTPS, FTP, SSH, SMTP, Database DNS, DHCP, VoIP, Video streaming, Gaming

TCP FLAGS:
  SYN = Synchronize sequence numbers (connection request)
  ACK = Acknowledgment

VLAN Configuration Example

Cisco 
# Create VLANs
Switch(config)# vlan 10
Switch(config-vlan)# name Sales
Switch(config)# vlan 20
Switch(config-vlan)# name IT
Switch(config)# vlan 30
Switch(config-vlan)# name Management

# Configure Access Port
Switch(config)# interface GigabitEthernet0/1
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 10

# Configure Trunk Port
Switch(config)# interface GigabitEthernet0/24
Switch(config-if)# switchport trunk encapsulation dot1q
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk native vlan 99
Switch(config-if)# switchport trunk allowed vlan 10,20,30

# Verify
Switch# show vlan brief
Switch# show interfaces trunk
HP/Aruba 
# Create VLANs
Switch(config)# vlan 10 name "Sales"
Switch(config)# vlan 20 name "IT"
Switch(config)# vlan 30 name "Management"

# Configure Access Port
Switch(config)# vlan 10 untagged 1-12
Switch(config)# vlan 20 untagged 13-24

# Configure Trunk Port
Switch(config)# vlan 10,20,30 tagged 25-26

# Verify
Switch# show vlans
Switch# show vlans ports

NAT Configuration Examples

Cisco 
# Static NAT
Router(config)# ip nat inside source static 192.168.1.10 203.0.113.10

# PAT (Overload) - Most common for internet access
Router(config)# access-list 1 permit 192.168.1.0 0.0.0.255
Router(config)# ip nat inside source list 1 interface GigabitEthernet0/1 overload

# Define inside and outside interfaces
Router(config)# interface GigabitEthernet0/0
Router(config-if)# ip nat inside
Router(config)# interface GigabitEthernet0/1
Router(config-if)# ip nat outside

# Verify NAT
Router# show ip nat translations
Router# show ip nat statistics
Router# clear ip nat translation *

Essential Protocols

ARP (Address Resolution Protocol)

ARP maps IP addresses (Layer 3) to MAC addresses (Layer 2) within a local network.

Bash 
# View ARP table
$ arp -a
$ ip neigh show

# Add static ARP entry
$ sudo arp -s 192.168.1.100 aa:bb:cc:dd:ee:ff

# Clear ARP cache
$ sudo ip neigh flush all

# Windows
C:\> arp -a
C:\> arp -d *  (clear cache)

DHCP Process (DORA)

DHCP DORA Process 
    DHCP Client                                         DHCP Server
         │                                                   │
         │──────── DISCOVER (Broadcast) ────────────────────►│
         │         "I need an IP address"                    │
         │                                                   │
         │◄─────── OFFER (Unicast/Broadcast) ────────────────│
         │         "Here's 192.168.1.100 for you"            │
         │                                                   │
         │──────── REQUEST (Broadcast) ─────────────────────►│
         │         "I'll take 192.168.1.100"                 │
         │                                                   │
         │◄─────── ACK (Unicast/Broadcast) ──────────────────│
         │         "Confirmed, it's yours for 8 hours"       │
         │                                                   │

  DHCP Provides:
  • IP Address
  • Subnet Mask
  • Default Gateway
  • DNS Servers
  • Lease Duration
  • Domain Name
